Q1. What is a shell script?  
A: A file containing Linux commands executed by a shell.

Q2. What is the purpose of the shebang line?  
A: It specifies which interpreter should execute the script.

Q3. What does #!/bin/bash indicate?  
A: The script should be run using the Bash shell.

Q4. Why is chmod +x required for scripts?  
A: To grant execute permission so the script can run.

Q5. How do you run a script from the current directory?  
A: Using ./script_name.sh.

Q6. What command was used to print output in the script?  
A: echo.

Q7. How can you verify script permissions?  
A: Using ls -l.

Q8. What happens if execute permission is missing?  
A: The script cannot be run as a program.

Q9. Why are shell scripts important for administrators?  
A: They automate repetitive tasks and workflows.

Q10. Where are basic shell scripts commonly used?  
A: Automation, monitoring, backups, and security operations.
