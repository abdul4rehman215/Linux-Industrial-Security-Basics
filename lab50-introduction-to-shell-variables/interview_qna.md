Q1. What is a shell variable?
A: A named value stored in the shell for use by commands and scripts.

Q2. How do you create a local shell variable?
A: MYVAR="value".

Q3. What is the difference between local and global variables?
A: Global variables are exported and accessible to child processes.

Q4. How do you export a variable?
A: Using export VAR=value.

Q5. What does the $? variable represent?
A: Exit status of the last executed command.

Q6. What does $$ represent?
A: Process ID of the current shell.

Q7. What does $# represent?
A: Number of arguments passed to a script or shell.

Q8. Why are environment variables security-sensitive?
A: They can expose secrets to other processes.

Q9. Are shell variables case-sensitive?
A: Yes.

Q10. Why are shell variables important in scripting?
A: They enable dynamic behavior, logic, and automation.
