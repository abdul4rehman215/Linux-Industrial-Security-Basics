Q1. What is the purpose of the touch command in Linux?  
A: It creates an empty file or updates timestamps of existing files.

Q2. How do you confirm that a file was created successfully?  
A: By checking its presence using the ls command.

Q3. What does the rm command do?  
A: It permanently deletes files from the filesystem.

Q4. Why should rm be used cautiously?  
A: Deleted files cannot be recovered, especially when using wildcards.

Q5. How do you verify a file exists before deleting it?  
A: By running ls filename.

Q6. What is the difference between cat and less?  
A: cat prints the entire file, while less allows paginated viewing.

Q7. When should less be preferred over cat?  
A: When viewing large files.

Q8. How was a file with content created in this lab?  
A: Using output redirection with echo.

Q9. Why are file management commands important for administrators?  
A: They are used to manage logs, configs, scripts, and system files.

Q10. Give a real-world use case for rm, cat, and less.  
A: Log inspection, config review, and cleanup of temporary files.
