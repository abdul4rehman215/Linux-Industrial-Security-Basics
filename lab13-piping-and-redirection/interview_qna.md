Q1. What is output redirection in Linux?  
A: Redirecting command output from the terminal to a file.

Q2. What does the > operator do?  
A: Redirects output to a file and overwrites existing content.

Q3. How do you append output to a file?  
A: Using the >> operator.

Q4. What is piping in Linux?  
A: Sending output of one command as input to another.

Q5. How was piping used in this lab?  
A: ps aux | grep bash.

Q6. Why is piping useful?  
A: It enables efficient filtering and processing of command output.

Q7. What happens if output.txt already exists when using >?  
A: The file is overwritten.

Q8. Why is >> safer than > in some cases?  
A: It preserves existing data.

Q9. Where are piping and redirection commonly used?  
A: Log analysis, scripting, monitoring, automation.

Q10. Why are these concepts important for Linux professionals?  
A: They allow powerful one-line workflows without complex scripts.
