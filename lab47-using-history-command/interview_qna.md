Q1. What is the purpose of the history command?
A: It displays previously executed shell commands.

Q2. Where is Bash history stored?
A: In memory during a session and written to ~/.bash_history.

Q3. What does each history entry contain?
A: A sequence number and the executed command.

Q4. How do you re-run a command from history?
A: Using !<number>.

Q5. Does re-running a command add it to history?
A: Yes, it becomes a new history entry.

Q6. What does history -c do?
A: Clears the current shell history.

Q7. Is history cleared permanently with history -c?
A: No, only the in-memory history is cleared.

Q8. Why might administrators clear history?
A: To remove sensitive or confidential commands.

Q9. How does history improve productivity?
A: It avoids retyping long or complex commands.

Q10. Why is history important for auditing?
A: It helps track previously executed actions.
