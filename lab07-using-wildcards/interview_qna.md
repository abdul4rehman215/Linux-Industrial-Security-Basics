Q1. What are wildcards in Linux?  
A: Special characters used to match multiple filenames.

Q2. What does the * wildcard represent?  
A: It matches any number of characters.

Q3. How was * used in this lab?  
A: To copy all .txt files using cp *.txt backup/.

Q4. Why are wildcards useful?  
A: They enable bulk operations without typing every filename.

Q5. What does the ? wildcard match?  
A: Exactly one character in a filename.

Q6. Which files were deleted using rm file?.txt?  
A: file1.txt and file2.txt.

Q7. Why were report files not deleted?  
A: They did not match the wildcard pattern.

Q8. What precaution should be taken when using wildcards with rm?  
A: Verify patterns with ls before deleting files.

Q9. Where are wildcards commonly used in real systems?  
A: Log cleanup, backups, automation scripts.

Q10. How do wildcards help system administrators?  
A: They save time and reduce manual errors.
