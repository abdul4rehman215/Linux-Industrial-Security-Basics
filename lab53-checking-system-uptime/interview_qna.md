Q1. What does the uptime command show?
A: System runtime, active users, and load averages.

Q2. What is system uptime?
A: The total time the system has been running since the last reboot.

Q3. What are load averages?
A: Metrics showing system load over 1, 5, and 15 minutes.

Q4. What does a low load average indicate?
A: The system is lightly loaded or idle.

Q5. How do you judge overload using load average?
A: Compare load average with CPU core count.

Q6. What does the first load average value represent?
A: System load over the last 1 minute.

Q7. What command shows CPU core count?
A: nproc.

Q8. Is a short load spike a problem?
A: No, short spikes are normal.

Q9. Why is uptime useful for administrators?
A: It provides a quick health and stability check.

Q10. When should high load be investigated?
A: When it remains high for long periods.
